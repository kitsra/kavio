Kavio API overview.

Kavio is split into focused packages so each layer has a clear contract: data, timeline logic, authoring, preview, render orchestration, media planning, and CLI.

Package Contracts

@kavio/schema
JSON Schema, shared types, validation, and migration scaffold.
@kavio/core
Timeline evaluation, frame math, easing, prop resolution, layout, captions, and resource limits.
@kavio/builder
TypeScript authoring helpers that output canonical Kavio JSON.
@kavio/browser-renderer
Browser preview and DOM rendering runtime.
@kavio/render-worker
Browser-driver contracts, frame capture helpers, batch expansion, metadata, and cleanup primitives.
@kavio/ffmpeg
Inspectable FFmpeg plan construction.
@kavio/cli
Local commands for validation, inspection, migration, and preview.
@kavio/render
Render execution layer that owns real browser capture, FFmpeg orchestration, MP4 output, metadata, cleanup, and cancellation.
@kavio/mcp
MCP server and provider tool adapters for agent-facing Kavio workflows. It calls @kavio/render instead of spawning binaries itself.

Import Style

Workspace examples import package helpers directly by package name after the monorepo is built.

import { prop, text, video } from "@kavio/builder";
