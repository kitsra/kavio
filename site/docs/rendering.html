Rendering is wired end to end.

kavio render selects the fast FFmpeg-direct path for eligible image sequences and keeps browser-overlay rendering as the full-fidelity path for richer compositions.

Implemented Pieces

  • Schema validation for composition documents.
  • Timeline evaluation and frame math in the core package.
  • Browser preview runtime for visual inspection.
  • Render-worker contracts, frame capture helpers, batch expansion, metadata, and cleanup primitives.
  • Inspectable FFmpeg plan construction.
  • Playwright-backed browser capture, FFmpeg execution, and CLI rendering.
  • Batch workers reuse Chromium processes while isolating every job in a fresh context and harness.
  • Deterministic PNG-sequence directories with zero-based frame names.
  • Thresholded SSIM/PSNR video and selected-frame comparison gates.
  • Finite audio looping and FFmpeg sidechain ducking.
  • MVP demo rendering through the shared render pipeline.

Intended Pipeline

  1. Validate the composition and prop rows.
  2. Expand template rows across export presets into stable jobs.
  3. Capture frames through the browser renderer.
  4. Build an inspectable FFmpeg plan.
  5. Encode media outputs and write metadata.
  6. Clean temporary files according to job policy.

Outputs And Quality

The current render path supports mp4, webm, mov, gif, still-image png, and png-sequence output. PNG sequences stream browser captures to a fresh directory as frame-00000.png, frame-00001.png, and so on. Use the comparison helper to gate full-video and selected-frame SSIM/PSNR before accepting a visual change. Browser-overlay remains the recommended mode for complex transitions, text, masks, and arbitrary keyframes. In batches, later compatible browser-overlay jobs reuse worker-local Chromium processes; timings.browserLaunches reports 0 when that reuse occurs. Within a loaded composition, the browser renderer also reuses deterministic plain-text, image, and shape DOM layers. Videos, captions, animated or keyframed layers, and transition participants remain frame-dynamic, and composition reloads clear the reused layer state.
