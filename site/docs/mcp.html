Kavio MCP server.

The MCP server is the agent-facing layer for Kavio. It exposes safe composition validation, inspection, prop resolution, render planning, rendering, resources, prompts, and provider tool adapters without making each agent integrate the video engine directly.

Status

The @kavio/mcp package is implemented. It calls @kavio/render for real render work and keeps Playwright, FFmpeg binaries, and process-spawning code out of MCP handlers.

Architecture Boundary

@kavio/mcp
Agent-facing protocol layer that exposes tools and resources for Kavio workflows.
@kavio/render
Execution layer for browser capture, FFmpeg orchestration, MP4 output, metadata, cleanup, and cancellation.
@kavio/schema
Composition validation and versioned data contract.
@kavio/core
Pure timeline evaluation, prop resolution, export presets, dimensions, and resource limits.
@kavio/browser-renderer
Shared render harness and browser runtime used for preview and export parity.

Tools

The MCP catalog exposes these workflows so agents can build and repair Kavio compositions reliably.

validate_compositionReturn structured schema, prop, timing, asset, and export errors. inspect_compositionSummarize duration, dimensions, layers, assets, audio, and export presets. migrate_compositionNo-op current schema documents and fail clearly for unsupported versions. resolve_propsResolve template placeholders from supplied prop values. list_export_presetsReturn standard reels, square, and landscape presets. plan_renderExpand jobs and assemble FFmpeg arguments without touching binaries. renderCall the render package for local video output when binaries are available.

Error Contract

MCP responses should preserve Kavio's structured error shape so agents can repair documents without scraping prose. 

{
  "code": "SCHEMA_VALIDATION_FAILED",
  "severity": "error",
  "message": "Layer duration exceeds composition duration.",
  "path": "/layers/2/durationFrames",
  "stage": "validate",
  "hint": "Reduce durationFrames or extend the composition.",
  "retryable": false
}

Safety And Supply Chain

  • Keep binary execution quarantined in @kavio/render.
  • Do not let MCP handlers spawn FFmpeg, launch browsers, or download assets outside the approved render layer.
  • Return structured validation errors before any render work begins.
  • Thread cancellation through long-running preview and render operations.
  • Respect the repository package-manager age gate for any dependency changes.
  • Keep default verification binary-free; put real render smoke tests behind a gated job.

Agent Workflow

  1. Generate or update a Kavio composition.
  2. Validate the composition and collect all repairable errors.
  3. Inspect the document to confirm dimensions, duration, assets, layers, audio, and exports.
  4. Preview locally for layout and timing checks.
  5. Render through @kavio/render.
  6. Return output metadata, checksums, and any structured errors to the agent.
